This project implements a Distributed Clock Synchronization System using UDP in Python. The objective of the project is to synchronize the clocks of multiple clients with a central time server in a distributed environment. In distributed systems, each machine maintains its own local clock, which may differ due to clock drift and network delays. To address this issue, a client-server architecture is used where the server acts as a reference clock and clients request the current time. The communication between the client and server is performed using the UDP protocol to ensure faster transmission with minimal overhead. When a client sends a synchronization request, it records the local timestamp, receives the server time, and calculates the Round Trip Time (RTT). The client then estimates the one-way network delay by dividing RTT by two and adjusts its clock using Cristian’s algorithm.

The system consists of three main modules: server, client GUI, and performance testing. The server listens for incoming client requests and responds with the current timestamp while also simulating real-world conditions such as processing delay and packet loss. The client GUI provides a user-friendly interface that displays local time, server time, adjusted time, simulated delay, and synchronization difference. It also implements timeout and retry mechanisms to handle packet loss in UDP communication. The performance testing module simulates multiple clients using multithreading to evaluate system behavior under concurrent requests and measure synchronization performance.

Additional improvements include packet loss handling, retry mechanism, server-side delay simulation, request logging, and multithreading for concurrent client execution. These enhancements make the system more realistic and fault-tolerant. Overall, the project demonstrates key networking and distributed system concepts such as UDP communication, socket programming, client-server architecture, RTT calculation, clock synchronization, threading, and reliability handling over an unreliable protocol.
